# 麦克斯韦妖 人工验收与探索性测试清单 (Checklist)

> **版本**: 1.0  
> **日期**: 2026-05-28  
> **说明**: 本文档用于指导测试人员在真实物理环境中进行自动化测试无法覆盖的“黑盒测试”、“边界破坏测试”及“UI/UX 体验评估”。

---

## 阶段 1：物理环境与真实文件系统破坏演习

本阶段旨在验证底层引擎在面对真实的、不可控的物理级文件系统波动时的鲁棒性。

### 1.1 大数据量抗压 (Big Data Stress)
- [ ] **准备海量细碎文件**：放入 10 万个以上的小文件（如 node_modules 或海量缩略图），测试阶段 1 遍历时内存是否爆掉。
- [ ] **准备极端大文件**：准备几个 50GB 以上的超大视频文件（部分相同、部分不同），测试阶段 2 的流式 SHA-256 计算是否引发 NAS CPU 持续 100% 满载卡死。
- [ ] **大量冗余组渲染**：构造产生 5,000 个以上重复组的结果。进入“分类明细”页，用力滚动鼠标滚轮。
  - **验收标准**：滑动时不发生严重的幻灯片卡顿（若极度卡顿，则证明必须要引入前端虚拟滚动）。

### 1.2 物理级破坏与状态偏移
- [ ] **幽灵文件 (Ghost Files)**：在生成清理方案后（此时结果已在前端渲染），使用操作系统的资源管理器或 NAS 后台手动删掉其中某个待清理文件。切回 麦克斯韦妖 点击“移动到废纸篓”。
  - **验收标准**：应当仅报告该文件移动失败并跳过，其余正常文件正常清理，不能弹 500 崩溃页面。
- [ ] **文件深度锁定 (Lock Contention)**：把其中一个重复文件用 Premiere/Word 等软件打开不关。在 麦克斯韦妖 中尝试将其移动。
  - **验收标准**：正确捕获“文件被占用”异常并优雅提示。
- [ ] **外置盘掉线 (Mount Disconnect)**：将扫描目录设为一个 U 盘或外挂存储，扫描完成准备移动时，强行拔掉 U 盘。点击清理。
  - **验收标准**：优雅捕获路径丢失或 I/O 错误。

## 阶段 2：网络断层与通信恢复测试 (Network & WS)

针对基于 WebSocket 的进度推送体系的弱网防御。

### 2.1 长连接健壮性
- [ ] **扫描中途断网重连**：启动耗时较长的扫描，断开电脑 Wi-Fi 5 秒后再连上。
  - **验收标准**：WebSocket 断开后，页面能否依赖定时轮询（Polling fallback）兜底进度，或提示用户网络异常，但不至于丢失会话 ID。
- [ ] **休眠唤醒测试**：开始执行批量移动操作，合上笔记本盖子休眠。5 分钟后翻开。
  - **验收标准**：此时操作大概率已经在后端完成。前端页面应该能够自动刷新状态，或引导用户回到首页查看结果。

### 2.2 暴力并发测试 (Monkey Click)
- [ ] **疯狂点击狂人**：在“新建扫描”页，以极快速度连击“开始扫描”按钮 10 次。
  - **验收标准**：防抖或防重入机制生效，只创建并跳转一次扫描任务，且后端正确拒绝并发请求（API-08 生效）。
- [ ] **疯狂状态切换**：在分类明细页，对着一个文件的“保留/删除”勾选框疯狂连点。
  - **验收标准**：状态不能因为后端异步延迟而产生状态倒错（UI 状态与后端最终落库状态脱节）。

## 阶段 3：UI/UX 与前端视觉边界盲区

关注视觉呈现和操作合理性。

### 3.1 极值视觉溢出
- [ ] **超长深渊路径**：制造一段超过 150 个字符的变态文件树层级，且包含 emoji（如 `📁第一层/📁第二层/📁.../📁最后一层🎈/超长文件名字.txt`）。
  - **验收标准**：列表、进度条下方以及标记树等地方的文字必须能优雅地截断（`text-overflow: ellipsis`）并支持 hover 显示全称，绝对不能把容器宽向撑破破坏布局。
- [ ] **0 KB 可释放空间**：当扫描到的都是 0 字节（空）文件的重复时。
  - **验收标准**：UI 显示“可释放 0.00 MB”或类似提示时不能出现 NaN 或排版崩塌。

### 3.2 认知盲区与交互友好度
- [ ] **置灰反人类测试**：故意制造一个“全删除组”（触发红线），此时清理按钮置灰。找一个不熟悉系统规则的非开发人员。
  - **验收标准**：观察他能否通过 UI 上的提示（比如“该组全部被删除，为防误删已锁定操作”之类的警告语）明白为什么按钮点不了，以及如何去修复（改一个文件为 keep）。如果他完全懵逼，说明还需要加浮窗提示。
- [ ] **树形标记全选与折叠迷失**：展开所有目录（几百个），标记了一堆。然后点“收起子目录”。
  - **验收标准**：收起后，上层父目录是否能清晰传达“我的内部有文件被标记了”的视觉信息？如果没有，用户可能忘了刚才标记过啥。

## 阶段 4：跨平台系统兼容性走查

- [ ] **Linux (NAS 容器)** vs **Windows (原生)**：
  - NAS 模式下验证移动到 `#recycle` 是否符合群晖用户的操作直觉。
  - Windows 原生模式下验证是否正确挂载盘符（如 `D:\`、`E:\`）并能正常使用系统自带回收站。
- [ ] **移动端兼容**：
  - 使用 iPad 或大屏手机打开后台页面。
  - **验收标准**：尽管该工具偏向 PC 操作，但核心功能不应发生严重阻断性错位，能勉强完成查看与简单处理。
